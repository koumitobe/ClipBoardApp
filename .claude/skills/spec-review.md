# Skill: 仕様書レビュー

> このSkillは仕様書の品質チェックを行う再利用可能なプロセス定義です。

## 使用場面

- 新しい仕様書が作成・更新されたとき
- `/spec-check` コマンドから呼び出されるとき

## チェックリスト

### 要件定義書（PRD）

**完全性チェック**
- [ ] 背景・目的が明確か
- [ ] スコープのIn/Out Scopeが定義されているか
- [ ] ユーザーストーリーが存在するか
- [ ] 全機能要件に受け入れ条件があるか
- [ ] 非機能要件（パフォーマンス・セキュリティ等）が定義されているか
- [ ] 用語定義が存在するか

**品質チェック**
- [ ] 曖昧な表現（「適切に」「なるべく」等）が使われていないか
- [ ] 矛盾する要件がないか
- [ ] 実現可能性の高い要件か

### 設計書（DS）

- [ ] PRDの全機能要件が設計に反映されているか
- [ ] データモデルが定義されているか
- [ ] エラーハンドリングが定義されているか
- [ ] セキュリティ考慮事項が記載されているか

### API仕様

- [ ] リクエスト・レスポンスのサンプルがあるか
- [ ] バリデーションルールが定義されているか
- [ ] エラーレスポンスが定義されているか
- [ ] 認証方式が明記されているか

## 報告フォーマット

問題を発見した場合は以下の形式で報告する:

```
## 仕様書レビュー結果: <ファイル名>

### 重大な問題（要修正）
- [行XX] 受け入れ条件が未定義: FR-003

### 軽微な問題（推奨修正）
- 曖昧な表現: 「適切に処理する」→ 具体的な処理内容を記述してください

### 提案
- FR-002とFR-004は類似しており、統合を検討してください
```
