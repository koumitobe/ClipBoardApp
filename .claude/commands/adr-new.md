# /adr-new — ADR新規作成

## 目的
アーキテクチャ上の意思決定を記録するADRファイルを作成する。

## 使い方

```
/adr-new <決定事項のタイトル>
```

## 実行手順

1. `docs/adr/` 内の最新ADR番号を確認する
2. 次の番号で `docs/adr/ADR-XXX_<title>.md` を作成する
3. ユーザーに以下をヒアリングする:
   - 背景・コンテキスト
   - 検討した選択肢（最低2つ）
   - 採用した決定と理由

4. `docs/adr/ADR-000_template.md` をベースにファイルを生成する

## ADR番号体系

- ADR-001〜099: インフラ・基盤
- ADR-100〜199: アーキテクチャ・設計
- ADR-200〜299: 技術選定
- ADR-300〜399: プロセス・開発ルール

## 注意事項

- 一度AcceptedになったADRは削除しない
- 方針変更時は新しいADRを作成し、古いADRを "Superseded by ADR-XXX" にする
- ADRは決定の記録であり、設計書ではない
