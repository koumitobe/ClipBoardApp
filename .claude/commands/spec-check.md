# /spec-check — 仕様整合性チェック

## 目的
現在の実装が仕様書と整合しているかを確認し、乖離点を報告する。

## 実行手順

1. `specs/requirements/` の全PRDファイルを読み込む
2. `specs/design/` の設計書を読み込む
3. `src/` の実装を走査する
4. 以下の観点でチェックを行う:

### チェック観点

**仕様カバレッジ**
- PRDに記載された機能要件が実装されているか
- 未実装の "Must" 要件がないか

**仕様逸脱**
- 仕様に記載のない機能が実装されていないか
- APIのインターフェースが `specs/api/` と一致しているか

**データモデル整合**
- DBスキーマが `specs/design/` のデータモデルと一致しているか

## 出力フォーマット

```
## 仕様整合性チェック結果

### ✅ 適合項目
- FR-001: ログイン機能 — 実装済み

### ⚠️ 要確認項目
- FR-003: パスワードリセット — 仕様あり、実装未確認

### ❌ 乖離・未実装
- FR-002: ソーシャルログイン — Must要件だが未実装

### 📝 仕様外実装
- src/utils/debug.ts — 仕様に記載なし
```
