# CLAUDE.md — AI行動指針

> このファイルはClaudeCodeが読み込む最上位の行動ルールです。
> プロジェクト固有の詳細は各ディレクトリの CLAUDE.md に記述してください。

---

## プロジェクト概要

- **プロジェクト名**: `ClipBoardApp`
- **目的**: Macのクリップボード履歴を最大30件保持し、Commandキー2回押しでマウス付近にポップアップ表示して過去のコピー内容を呼び出せるツール
- **技術スタック**: Swift / SwiftUI / AppKit / DMG配布

---

## 基本ルール

### 仕様駆動の原則
1. **仕様を先に読む**: コードを書く前に必ず `specs/` を確認する
2. **仕様と実装の乖離を報告**: 実装が仕様と異なる場合は即座に指摘する
3. **仕様外の実装は禁止**: 仕様に記載のない機能を勝手に追加しない
4. **変更は仕様から**: 実装の変更前に対応する仕様ファイルを更新する提案をする

### コミュニケーションルール
- 不明点は実装前に必ず確認する
- 複数の実装方針がある場合は選択肢を提示する
- 破壊的変更を行う前に必ず警告する
- 長いタスクは `tasks/` にチェックリストを作成してから着手する

### コーディングルール
- コードには日本語コメントを付与する（コメント言語: 日本語）
- 関数・クラスにはJSDocまたは同等のドキュメントを付ける
- テストなしの実装PRは原則禁止
- マジックナンバーは定数化する
- 極力1ファイル20行以内に収める努力を行い難しい場合には関数化などを行い可読性を上げる

## ディレクトリ構成と役割

```
project-root/
├── CLAUDE.md              # このファイル（AI行動指針）
├── .claude/               # ClaudeCode設定
│   ├── commands/          # カスタムスラッシュコマンド
│   ├── skills/            # 再利用可能なSkill定義
│   └── settings.json      # ClaudeCode設定
├── specs/                 # 仕様書（最重要・変更時は要確認）
│   ├── requirements/      # 要件定義
│   ├── design/            # 設計仕様
│   ├── api/               # API仕様
│   └── changelog/         # 仕様変更履歴
├── docs/                  # 技術ドキュメント
│   ├── adr/               # Architecture Decision Records
│   ├── runbooks/          # 運用手順書
│   └── architecture.md    # システム構成図・説明
├── src/                   # 実装コード
├── tests/                 # テストコード
├── tasks/                 # タスク・進捗管理
└── .env.example           # 環境変数テンプレート
```

---

## タスク管理ルール

- 中規模以上のタスク（30分以上）は `tasks/` にMarkdownファイルを作成
- タスクファイルは `tasks/YYYYMMDD_task-name.md` の命名規則
- チェックリスト形式で進捗を管理する
- 完了タスクは `tasks/done/` に移動する

---

## 禁止事項

- `specs/` ディレクトリの無断変更
- `.env` ファイルのコミット
- `console.log` のコミット（デバッグログ除く）
- テストなしでのコア機能実装

---

## 参照ファイル優先順位

1. `specs/requirements/` — 何を作るか
2. `specs/design/` — どう作るか
3. `docs/adr/` — なぜそう決めたか
4. `src/` — 実際の実装
